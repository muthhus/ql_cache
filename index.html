ql_cache

A simple wrapper that makes localStorage and JSON play well together.

The full readme is on Github. If you're not familiar with localStorage, be sure to check out the w3c spec and its section in Dive into HTML5.

This option gives you a non-destructive way of saving multiple values into a single key. LocalStorage is currently capped at 5mb for most browsers, so ql_cache limits how many values you can store in a single key. You can override the limit by passing a number as the value.

One of the best use cases for localStorage is saving JSON from AJAX responses. To help with knowing when you need to refresh data, ql_cache gives you the ability to store an "expires" timestamp when you save a value into storage. Given the key/value nature of localStorage, it's easier to store the expires timestamp in a separate key, following the pattern of "key name" + "expires".
